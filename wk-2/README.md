# Task API

Task API is a small FastAPI project that demonstrates CRUD with an in-memory list. It includes filtering, search, stats, and reset endpoints, with Swagger UI at `/docs`.

## Installation guide

### 1) Clone the repository

```bash
git clone https://github.com/williskipsjr/flyrank-sandbox.git
cd flyrank-sandbox/wk-2
```

### 2) Create and activate a virtual environment

Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

macOS/Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3) Install dependencies

```bash
pip install fastapi uvicorn
```

### 4) Start the API server

```bash
uvicorn main:app --reload
```

### 5) Open in browser

- `http://127.0.0.1:8000/`
- `http://127.0.0.1:8000/docs`

## How to run

1. Activate your virtual environment.
2. Start the server from the `wk-2` folder:

```bash
uvicorn main:app --reload
```

3. Open:
   - `http://127.0.0.1:8000/`
   - `http://127.0.0.1:8000/docs`

## Quick demo flow (copy/paste)

Use this exact sequence to demo the API quickly from terminal.

```bash
# 1) API info
curl -X GET "http://127.0.0.1:8000/"

# 2) Health check
curl -X GET "http://127.0.0.1:8000/health"

# 3) List starter tasks
curl -X GET "http://127.0.0.1:8000/tasks"

# 4) Create a task
curl -X POST "http://127.0.0.1:8000/tasks" -H "Content-Type: application/json" -d "{\"title\":\"Practice FastAPI\"}"

# 5) Mark task 1 done and rename it
curl -X PUT "http://127.0.0.1:8000/tasks/1" -H "Content-Type: application/json" -d "{\"title\":\"Buy milk and bread\",\"done\":true}"

# 6) Filter done tasks
curl -X GET "http://127.0.0.1:8000/tasks?done=true"

# 7) Search by title
curl -X GET "http://127.0.0.1:8000/tasks?search=milk"

# 8) See computed stats
curl -X GET "http://127.0.0.1:8000/stats"

# 9) Delete one task
curl -X DELETE "http://127.0.0.1:8000/tasks/3"

# 10) Reset demo data to starter tasks
curl -X POST "http://127.0.0.1:8000/reset"

# 11) Confirm reset worked
curl -X GET "http://127.0.0.1:8000/tasks"
```

## Mortality experiment

I created extra tasks, restarted the server, and called `GET /tasks`; only the starter tasks were present again.
This happens because data is stored only in memory (RAM), and process memory is cleared when the server stops.
